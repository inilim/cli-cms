# Правило использования статических анонимных функций

## Принцип
Если в анонимной функции не требуется контекст объекта (то есть не используется `$this`), то всегда используйте статические анонимные функции с ключевым словом `static`.

## Примеры

### Правильно:
```php
$categoryIds = \array_filter($categoryIds, static fn($id) => $id !== null);
```

### Неправильно:
```php
$categoryIds = \array_filter($categoryIds, fn($id) => $id !== null);
```

## Преимущества
- Улучшает производительность, так как не создает замыкание с контекстом объекта
- Явно указывает, что функция не зависит от контекста объекта
- Лучше отражает намерения разработчика
