# Web Content Accessibility Guidelines (WCAG) для CLI-CMS

## Цель
Обеспечение доступности веб-контента, генерируемого CLI-CMS, для всех пользователей, включая людей с ограниченными возможностями.

## Основные принципы WCAG 2.1

### 1. Доступность (Perceivable)
- Предоставление альтернативного текста для небольших изображений
- Создание подписей к видео и аудио контенту
- Использование семантически правильной разметки
- Обеспечение возможности адаптации контента

### 2. Управляемость (Operable)
- Обеспечение навигации с клавиатуры
- Предоставление достаточного времени для чтения и использования контента
- Предотвращение контента, вызывающего судороги
- Обеспечение удобства навигации

### 3. Понятность (Understandable)
- Использование понятного языка
- Предсказуемое поведение веб-страниц
- Введение вспомогательных инструкций при необходимости
- Четкая идентификация языка

### 4. Надежность (Robust)
- Совместимость с вспомогательными технологиями
- Правильная структура разметки
- Корректное использование ARIA-атрибутов

## Требования к разработке шаблонов CLI-CMS

### Семантическая разметка
- Использование заголовков (h1-h6) в логическом порядке
- Применение тегов article, section, nav, aside и других по назначению
- Корректное использование списков (ul, ol, dl) для структурирования информации

### Атрибуты доступности
- Атрибут `alt` для всех изображений с описательным текстом
- Использование `aria-label`, `aria-labelledby`, `aria-describedby` при необходимости
- Применение `role` атрибутов при необходимости (но с осторожностью)

### Цвет и контрастность
- Обеспечение минимального соотношения контрастности 4.5:1 для нормального текста
- Обеспечение минимального соотношения контрастности 3:1 для крупного текста
- Не использовать цвет как единственный способ передачи информации

### Формы и интерактивные элементы
- Корректные метки (label) для всех полей ввода
- Четкое визуальное выделение фокуса элементов
- Логичный порядок табуляции

## Проверка доступности

### Автоматизированные инструменты
- Использование инструментов вроде axe-core для проверки шаблонов
- Интеграция проверок в процесс разработки

### Ручная проверка
- Тестирование с клавиатуры без использования мыши
- Проверка контрастности цветов
- Тестирование с экранной читалкой

## Применение в CLI-CMS

### Шаблоны Twig
- При создании шаблонов в `files/templates/` учитывать требования WCAG
- Особое внимание к шаблонам блоков в `files/templates/blocks/`
- Проверка шаблонов для компонентов в `files/templates/components/`

### Генерация контента
- Обеспечение доступности при обработке JSON-контента из Editor.js
- Корректная конвертация структурированного контента в доступную разметку
- Поддержка альтернативного текста для медиа-элементов

### CSS-стили
- Создание стилей с учетом потребностей пользователей с различными нарушениями
- Поддержка масштабирования текста
- Обеспечение видимости индикатора фокуса

## Уровни соответствия

### Уровень A (Минимальный)
- Обязательный уровень для всех проектов CLI-CMS

### Уровень AA (Средний)
- Рекомендуемый уровень для большинства проектов CLI-CMS

### Уровень AAA (Высокий)
- По требованию для специализированных проектов

## Проверочный лист для шаблонов

- [ ] Используются семантически правильные HTML-теги
- [ ] Все изображения имеют атрибут `alt` с описательным текстом
- [ ] Заголовки расположены в логическом порядке (h1, h2, h3...)
- [ ] Цвет не используется как единственный способ передачи информации
- [ ] Соотношение контрастности текста соответствует нормам
- [ ] Все интерактивные элементы доступны с клавиатуры
- [ ] Есть видимый индикатор фокуса у активных элементов
- [ ] Формы имеют корректные метки
- [ ] Используются корректные ARIA-атрибуты при необходимости
- [ ] Структура страницы понятна и логична
