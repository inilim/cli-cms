# Использование DI-контейнера

## Правильное использование DI в проекте

В проекте используется библиотека `inilim/di` для внедрения зависимостей. Для получения экземпляров классов из DI-контейнера рекомендуется использовать следующие глобальные функции:

### Функция `DI()`
- Используется для получения экземпляра класса из DI-контейнера
- Пример: `\DI(ClassName::class)`
- Является наиболее лаконичным способом получения зависимостей

### Функция `DITag()`
- Используется для получения экземпляра по тегу из DI-контейнера
- Пример: `\DITag('config')`
- Подходит для получения объектов, зарегистрированных с тегами

### Через объект DI::self()
- Менее предпочтительный способ: `DI::self()->DI(ClassName::class)`
- Используется в тех случаях, когда нужно явно работать с экземпляром DI-контейнера

## Примеры использования

### В контроллерах:
```php
$recordRepository = \DI(RecordRepository::class);
$twigRenderService = \DI(TwigRenderService::class);
```

### При работе с тегами:
```php
$config = \DITag('config');
```

## Преимущества использования глобальных функций
- Более лаконичный и читаемый код
- Прямой доступ к зависимостям без необходимости явно работать с DI-контейнером
- Согласованность с архитектурными принципами проекта
